# SKN18-1st-5Team
# 🚗 전국 자동차 등록 및 현대자동차 판매 분석

## 📊 프로젝트 개요
본 프로젝트는 2017년부터 2024년까지의 전국 자동차 등록 현황과 2017년부터 2025년까지의 현대자동차 판매 데이터를 기반으로 구성된 **Streamlit 대시보드**입니다.  
사용자는 전국 자동차의 등록 현황을 알 수 있으며, 현대 자동차의 점유율 및 판매추세와 FAQ 시스템을 통해 차량 구매 시 궁금한 점도 함께 해결할 수 있습니다.

---

## 🚀 주요 기능

### 1. 전국 자동차 등록 현황
- 연도별 차량대수 증감률 분석
- 연도별 신차등록 추이 분석
- 시도/시군구별 차량 등록 현황 분석
- 성별 차량 등록 현황 분석
- 차종별 등록 현황 분석

### 2. 현대자동차 점유율 및 판매 추세
- 연도별 현대차 판매 추세 (2017~2025)
- 현대차의 시장 점유율 분석
- 국내 vs 해외 판매 비율 비교
- 상위 차종별 판매 비중 분석
- 연도별 판매 실적 증감률 계산

### 3. 현대자동차 FAQ 시스템
- 카테고리별 질문/답변 탐색
- 질문 제목 및 내용으로 검색
- 페이지네이션 지원
- 실시간 검색 기능 제공

## 📊 데이터베이스 스키마 (ERD 요약)

### 1. `integrated_statistics` (전국 자동차 등록 종합 통계)
- 성별(gender), 광역자치단체(city), 기초자치단체(district), 연도(year), 차종별 판매량(car_type), 신차등록(new_car), 차량 등록 대수(count)
- 각 테이블마다 정규화된 형태로 구성

### 2. `hyundai_sale` (현대차 판매 데이터)
- 연도별 모델별 판매 실적 기록
- 국내 판매(`domestic_count`), 해외 수출(`export_count`), 총합(`total_count`), 연도(`year`)정보 제공

### 3. `faq`, `faq_category`
- 현대차 관련 FAQ 시스템 구성
- 질문/답변 및 카테고리 분류 지원

## 💬 팀원별 느낀점

### 👤 팀장 - 최준호
> 이번 프로젝트를 진행하면서 강사님께서 강조하신 "설계"의 중요성을 많이 느꼈습니다.  웹 구현에 있어 LLM에 거의 의존하다보니, 페이지 이동에 문제가 생기다보니, 많은 어려움을 느꼈고, 다른 팀원의 구현내용을 조합하는 과정에서 이전에 발생하지 않았던 부분이 문제가 발생하는 경우가 있어서, 다음 프로젝트에서 구축 역활을 하게 된다면, 팀원의 결과물을 조합하는 상황을 염두해 두고, 미리 설계한 다음, 설계한 틀에서 LLM이나 팀원과의 조율을 통하여 조합하는 과정에서 오류가 발생하지 않도록 하는 것이 중요하다고 생각하였습니다.
